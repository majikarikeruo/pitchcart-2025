# PitchCart 再始動計画：継続的な改善サイクル実現への道筋

## 1. 目的

本計画の目的は、PitchCartを単発の分析ツールから、ユーザーのプレゼンテーションスキル向上に継続的に伴走する「パーソナルコーチ」へと進化させることである。迷走状態から脱却し、着実な一歩を踏み出すことで、プロジェクトの価値を再定義し、明確な進捗を生み出す。

## 2. 開発ロードマップ（ステップ・バイ・ステップ）

### Step 1: 【最優先】分析結果の履歴管理機能（プロトタイプ）

**目的:** ユーザーが過去の分析結果を振り返れるようにし、「改善の記録」という基本的な価値を提供する。

- **タスク:**
    1.  **データ構造の設計:** 複数の分析結果をバージョン管理できるJSON形式を定義する。（例: `analysisHistory.json`）
    2.  **UIの追加:** 結果ページ (`Result.tsx`) に、過去の分析バージョンを切り替えるためのシンプルなUI（ドロップダウンリスト等）を設置する。
    3.  **ロジックの実装:** 選択されたバージョンの分析結果を動的に表示するよう、データ取得フック (`useResults.ts`) を改修する。

### Step 2: バージョン比較機能

**目的:** 新旧の分析結果を並べて比較できるようにし、ユーザーが「改善による変化」を具体的に実感できるようにする。

- **タスク:**
    1.  **比較UIの設計:** 2つのバージョンのスコア、指摘事項などを並べて表示する画面をデザインする。
    2.  **差分ハイライト機能:** スコアの増減や、改善済み・新規発生した課題などを視覚的にハイライトするロジックを実装する。
    3.  **サマリー生成AI:** 「前回の指摘に基づき市場分析が改善され、スコアが15点向上しました」のような、変化を要約するテキストをAIで自動生成する機能を検討する。

### Step 3: 実践フィードバックの入力と活用

**目的:** 実際のプレゼン結果を記録し、それを次の分析に活かすことで、よりパーソナライズされたアドバイスを提供する。

- **タスク:**
    1.  **入力フォームの作成:** プレゼン後の結果（聴衆の反応、受けた質問など）を簡単に入力できるUIを作成する。
    2.  **データ保存:** 入力されたフィードバックを、分析履歴と紐付けてデータベースに保存する。
    3.  **プロンプトの動的化:** 保存されたフィードバックを、次回の分析時にAIへのプロンプトに含めることで、分析精度を向上させる。（例: 「前回のプレゼンでは〇〇という質問が多かった点を踏まえ、今回はその説明が十分か重点的に分析してください」）

### Step 4: 成長の可視化とゲーミフィケーション

**目的:** ユーザーが自身の成長を楽しみながら実感できるよう促し、継続的な利用を動機付ける。

- **タスク:**
    1.  **ダッシュボードの作成:** スコアの推移や改善項目の達成状況などをグラフで可視化するダッシュボードページを作成する。
    2.  **ゲーミフィケーション要素の導入:** 特定の課題をクリアすることでバッジを獲得できるなど、ゲーム感覚で取り組める仕組みを導入する。

## 3. 技術的な考慮事項

- **データ永続化:** ローカルストレージ（ブラウザ）から始め、将来的にはFirebase/Supabase等のBaaS（Backend as a Service）を活用し、ユーザー間でデータを扱えるように拡張することを見据える。
- **状態管理:** アプリケーションの状態が複雑化することに対応するため、状態管理ライブラリ（Zustand, Recoilなど）の導入を検討する。
- **コンポーネント設計:** 各機能（履歴、比較、ダッシュボード等）を再利用可能なコンポーネントとして設計し、保守性を高める。

この計画は、まずStep 1を確実に完了させることから始めます。小さな成功を積み重ねることが、プロジェクトを再び軌道に乗せるための最も確実な方法です。
